# 分类标签（目录结构）

- `00_索引/`：INDEX、摘要、TODO
- `10_周报与复盘/`：Wxx、复盘、规划
- `20_方法论_工作流/`：Copilot、Agent、Planning
- `30_行业_汽车/`：汽车策略、车型、销量监测
- `40_创作_公众号/`：选题、写作、标题、提纲、草稿、发布复盘

# 标准化命名规则

## 日期类

`YYYY-MM-DD 主题.md`

示例：`2025-06-18 项目进展.md`

兼容输入（会被规范化）：
- `0618 项目进展.md` → `2025-06-18 项目进展.md`
- `0709项目回顾.md` → `2025-07-09 项目回顾.md`

## 周报类

`YYYY-Wxx 主题.md`

示例：
- `W25 复盘.md` → `2025-W25 复盘.md`
- `W26 规划：一键报告.md` → `2025-W26 规划：一键报告.md`

## 系列类（示例：Copilot工作流）

优先推荐：`系列名_YYYY-MM-DD.md`

示例：
- `0714 Copilot工作流.md` → `Copilot工作流_2025-07-14.md`
- `0716 Copilot工作流.md` → `Copilot工作流_2025-07-16.md`

# YAML 头部字段（写入 Markdown）

写入在文件最顶部：

```yaml
---
category: 20_方法论_工作流
created_at: 2025-07-14 14:36
updated_at: 2025-07-16 10:40
---
```

说明：
- `updated_at`：每次整理都会刷新为文件系统 mtime
- `created_at`：若已有则保留；若没有则用“可推断日期（文件名日期）或 mtime”填充
